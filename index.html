Welcome to JAIL

Jail is Just Another Interpreted Language, and Uses Silly Terms. So if you're wondering why Jail is called Jail: it's a double-recursive acronym. Just one of the things the OLOCA made me do.. (If you don't know what the OLOCA is, look here).

The Jail project is hosted by SourceForge.

If you came here looking for a full-fledged, ready-to-deploy interpreted language, you're out of luck: have a look at Perl, Ruby, Python, or some other interpreted language in stead.

If you came here looking to join the effort of defining a new interpreted language with ambitous goals, you're exactly where you're supposed to be.


Intended audience

The intended audience of the Jail language is the part of humanity that wants to use a computer without knowing how it works, which (I believe) goes for the vast majority of people.

For most people, the Bash shell is too complicated; Perl, Ruby and Python are beyond reach; C and Pascal are way too complicated and C++ and Java are beyond their wildest dreams.

Jail is intended for those people for whom an accessible language is not available at this time.

Goals

The goals of the Jail project are:

  • To make a very portable interpreted language, that will run on
    • any Linux system (be it Red Hat, Mandrake, SuSE, Debian or other)
    • any modern Windows system except Windows CE (using Cygwin)
    • any BSD system (be it FreeBSD, OpenBSD, NetBSD or other)
    • any other POSIX system
  • To make a very easy interpreted language, that can be used by practically anyone
    • by clearly documenting every part of the language
    • by transparently converting from any scalar type to another
    • by handling all memory allocation internally
    • by handling pass-by-value and pass-by-reference in a "natural" way
  • To make a very "powerful" language
    • by being object-oriented all the way
    • by implementing a small but complete instruction set
    • by implementing integrated database access
    • by implementing Perl-compatible regular expressions
    • by implementing a simple way to make graphical user interfaces on top of X
  • To make a secure language
    • e.g. by making the interpreter configurable as to what access it should allow to system resources
      e.g. it should be possible to disallow file system access if the administrator doesn't want you to have it (e.g. for downloaded scripts embedded in HTML pages)
    • by taking security considerations into account from the very start of the language's design

How you can help

The language (Jail) and the programs that implement it are a work in progress. We need

  • People from the intended audience
    Meaning people who may not know the difference between "Word" and "Windows", may never have programmed anything in their lives but want their computers to simply behave.
    If you want to automate a task on your computer but you don't know how to do it - you're part of the intended audience!
  • More people from the intended audience
    Meaning people who know how to program, perhaps make a living doing so, but want to learn and help create a new programming language designed to
    • make object-oriented programming easy
    • make self-modifying code a piece of cake
    • be portable
    • be easy to learn
  • Yet more people from the intended audience
    i.e. anyone between the two beforementioned groups
  • Programmers who want to help make the preprocessor, compiler and/or virtual machine
  • Linguists who want to help define the language
  • Biologists who want to use the language for DNA sequence analysis
  • Lawyers who want to use the language to keep track of their bibliography
  • Managers and marketers who want to use it to manage databases
  • everybody else!

We need all those people to put their heads together and define the language we call Jail.
